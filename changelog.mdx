## Getting started

This changelog tracks all notable changes to your documentation site. Keep it updated as you add new features, fix issues, or make improvements.

### How to update this changelog

Follow these conventions when adding entries:

1. **Group by date** - Use month headers and specific dates
2. **Categorize changes** - Use categories like:
   - **New features** - Major new additions
   - **Improvements** - Enhancements to existing content
   - **Bug fixes** - Corrections and fixes
   - **Breaking changes** - Changes that require user action
3. **Be specific** - Describe what changed and why it matters
4. **Link to pages** - Reference relevant documentation pages

### Example entry format

```markdown
### February 1, 2026

**New features**
- Added [API authentication guide](/api-reference/authentication)
- Introduced webhook event documentation

**Improvements**
- Enhanced code examples with more languages
- Updated screenshots to reflect new UI

**Bug fixes**
- Fixed broken links in navigation
- Corrected typos in quickstart guide
```
