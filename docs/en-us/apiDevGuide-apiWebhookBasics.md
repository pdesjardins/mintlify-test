      <td className=""><div className="">Webhook</div></td>
      <td className=""><div className="">A Web-based callback that allows the Toast platform to push updates (also known as <em className="">webhook events</em> ) to webhook consumers, reducing or eliminating the need to poll a Toast API.</div></td>
    </tr>
    <tr className="">
      <td className=""><div className="">Webhook URL</div></td>
      <td className=""><div className="">The URL of a webhook consumer service that is capable of receiving webhook events from the Toast platform.</div></td>
    </tr>
    <tr className="">
      <td className=""><div className="">Event category</div></td>
      <td className=""><div className="">A Toast-defined classification for the type of events a webhook sends. For example, the <code className="">partners</code> event category sends webhook events related to the addition or removal of partner integrations to a restaurant location.</div></td>
    </tr>
    <tr className="">
      <td className=""><div className="">Event type</div></td>
      <td className=""><div className="">A Toast-defined name for a specific type of webhook event, for example, a <code className="">partner_added</code> or <code className="">partner_removed</code> event.</div></td>
    </tr>
    <tr className="">
      <td className=""><div className="">Event GUID</div></td>
      <td className=""><div className="">An identifier for the specific webhook event that is generated by the Toast platform.</div></td>
    </tr>
    <tr className="">
      <td className=""><div className="">Webhook subscription</div></td>
      <td className=""><div className="">Maintained by Toast support, a webhook subscription associates a webhook endpoint with an event category. This defines the type of webhook events that will be sent to the webhook endpoint. A webhook endpoint may be associated with multiple event categories.</div></td>
    </tr>
    <tr className="">
      <td className=""><div className="">Webhook secret</div></td>
      <td className=""><div className="">A secret passphrase generated when a webhook subscription is created. The passphrase is used to sign the contents of a message posted to the endpoint associated with the webhook subscription. See <a href="apiDevGuide-apiMessageSigning" className="">Message signing</a>. <br/> The secret key is unique for each webhook subscription within an environment. For example, a <code className="">stock</code> webhook subscription has a different secret key than a <code className="">partner</code> webhook subscription for the same integration partner and within the same environment. For more information about webhook subscriptions, see <a href="apiDevGuide-apiWebhookBasics" className="">Webhook basics</a>. For more information about Toast API environments, see <a href="apiDevGuide-apiEnvironments" className="">Environments</a>.</div></td>
    </tr>
